# LLM集成设计

## 十、LLM集成设计

### 10.1 系统架构

```
┌──────────────────────────────────────────────┐
│              前端（微信小游戏/H5）              │
│  - 渲染：文字为主 + 简单插图 + 地图            │
│  - 交互：选择式对话 + 策略面板                │
│  - 离线：本地缓存关键状态                     │
└──────────────────────┬───────────────────────┘
                       │ API
┌──────────────────────┴───────────────────────┐
│                游戏服务器                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │ 游戏逻辑  │ │ Agent系统 │ │ 状态存储  │     │
│  └──────────┘ └──────────┘ └──────────┘     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │ 事件引擎  │ │ LLM管理器 │ │ 调度器    │     │
│  └──────────┘ └──────────┘ └──────────┘     │
└──────────────────────┬───────────────────────┘
                       │
┌──────────────────────┴───────────────────────┐
│              LLM Provider                     │
│     Claude / GPT / 文心一言 / 本地模型         │
└──────────────────────────────────────────────┘
```

### 10.2 Prompt分层设计

**L1: 系统Prompt（固定）**
```
你是策略游戏"三界风云录"的AI叙事引擎。
世界观：[世界观简述]
角色：[当前Agent角色]
规则：只输出JSON，禁止输出思考过程
```

**L2: 场景Prompt（动态）**
```
当前世界状态：{world_state}
涉及角色：{character_profiles}
历史上下文：{relevant_history}
玩家状态：{player_state}
```

**L3: 任务Prompt（指令）**
```
任务：{task_description}
输入：{player_input}
约束：{constraints}
输出格式：{output_format}
```

### 10.3 LLM调用优化

```
分级模型：
  - 重大事件（战争/王位更替）：Claude/GPT-4
  - 普通事件（日常任务/交易）：GPT-3.5/文心
  - 简单响应（问候/确认）：本地小模型

缓存：
  - 相似场景复用生成结果
  - NPC常用回复模板化

批处理：
  - 多个NPC唤醒合并为一次调用
  - 预测可能的玩家选择提前生成
```
