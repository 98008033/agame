# Agame 核心玩法循环设计

> **核心理念**：玩家不是"看故事的人"，而是"活在故事里的人"。每天玩家有明确的"行动清单"，通过主动行动积累能力、建立关系、影响世界，而不是被动等待事件发生。

---

## 一、现有设计的问题诊断

### 1.1 当前设计的缺失

```
现有设计（PLAN/06）：

登录阶段（5-10分钟）：
  1. 读晨报 → 被动接收信息
  2. 查看个人事件 → 被动响应
  3. 做2-3个决策 → 被动选择
  4. 收取奖励 → 被动领取

深度游玩（每周30分钟）：
  经营、外交、事件链、探索...

问题诊断：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  ❌ 玩家像"观众"而非"演员"                                     │
│     所有内容都是世界生成，玩家只是选择响应                      │
│                                                              │
│  ❌ 没有"我要做什么"的明确驱动                                 │
│     玩家不知道自己的目标是什么                                 │
│                                                              │
│  ❌ 时间没有意义                                              │
│     没有"每天能做多少事"的约束                                 │
│     没有"这件事需要多久"的概念                                 │
│                                                              │
│  ❌ 行动缺乏反馈                                              │
│     做完一件事，看不到立刻的效果                               │
│     后果延迟到"几天后"才出现                                   │
│                                                              │
│  ❌ 缺乏日常积累感                                            │
│     除了事件，日常没有"我在变强"的感觉                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 参考优秀设计

| 游戏 | 值得借鉴的设计 | Agame如何借鉴 |
|------|---------------|---------------|
| **火焰纹章** | 策略时间分配，每个行动消耗时间资源 | 行动点系统：每天有限行动次数 |
| **Papers, Please** | 每日决策压力，做不完会有后果 | 每日任务压力：不处理的事件可能恶化 |
| **文明** | 回合行动选择，每个行动有明确效果 | 每个行动有即时反馈和可见效果 |
| **极乐迪斯科** | 对话选择深度，技能决定能看到什么 | 技能解锁新行动选项和叙事视角 |
| **CK3** | 人物行动清单，每个人都有明确可做的事 | 每日行动清单界面 |

## 二、每日行动清单系统

### 2.1 核心设计理念

```
玩家每天打开游戏，第一眼看到的是：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【今日计划】第15游戏日                                       │
│                                                              │
│  你有 6 个行动点可用                                          │
│                                                              │
│  ───────────────────────────────────────────────────────    │
│                                                              │
│  【日常行动】                                                 │
│  ├─ 练习技能 ────── 1AP ──── 武力+2经验                      │
│  ├─ 采集资源 ────── 2AP ──── 获得铁矿×3                      │
│  ├─ 打怪狩猎 ────── 3AP ──── 金币+50, 战斗经验                │
│  ├─ 社交拜访 ────── 1AP ──── 关系+5~10                       │
│  ├─ 学习技能 ────── 2AP ──── 需要导师或书本                   │
│  ├─ 工作赚钱 ────── 2AP ──── 金币+30~50                       │
│  └─ 帮助NPC ─────── 1AP ──── 关系+10，可能触发事件            │
│                                                              │
│  【事件响应】                                                 │
│  ├─ 处理待决事件 ─── 1-3AP ──── 取决于事件复杂度              │
│  ├─ 参与阵营会议 ── 2AP ──── 影响阵营决策                     │
│  └─ 响应NPC请求 ─── 1AP ──── 获得关系和可能的回报             │
│                                                              │
│  【待办提醒】                                                 │
│  ├─ ⚠️ 铁壁将军的请求将在3天后过期                            │
│  ├─ ⏰ 技能学习进度：情报分析 L2 → L3 (还差30经验)             │
│  └─ 📅 团体聚会将在明天举行                                   │
│                                                              │
│  今日已做：0/6AP                                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘

玩家看到这个界面就知道：
"今天我要练技能、拜访老锤、处理那个事件，还剩1AP存着备用。"

这就是"我要做什么"的明确驱动。
```

### 2.2 每日行动分类

**一、个人成长行动**

| 行动 | AP消耗 | 效果 | 条件 | 即时反馈 |
|------|--------|------|------|----------|
| **练习技能** | 1AP | 技能经验+2~5 | 需有该技能基础 | 显示进度条增长 |
| **学习技能** | 2AP | 新技能L1或+10经验 | 需导师/书本/事件解锁 | 显示技能解锁动画 |
| **闭关修炼** | 3AP | 技能经验+10~20 | 需要特定地点 | 显示修炼成果 |
| **阅读典籍** | 1AP | 学识+2，可能发现线索 | 需有书籍 | 显示发现的线索 |

**二、资源获取行动**

| 行动 | AP消耗 | 效果 | 条件 | 即时反馈 |
|------|--------|------|------|----------|
| **采集资源** | 2AP | 获得资源×3~5 | 需在资源区 | 显示获得物品清单 |
| **打怪狩猎** | 3AP | 金币+30~100，战斗经验 | 需有战斗能力 | 显示战果和物品 |
| **工作赚钱** | 2AP | 金币+30~50 | 需有职业身份 | 显示收入和评价 |
| **交易买卖** | 1AP | 商品流动，利润 | 需有商品/资金 | 显示交易结果 |
| **投资经营** | 2AP | 可能获得长期收益 | 需有资金和商机 | 显示投资状态 |

**三、社交行动**

| 行动 | AP消耗 | 效果 | 条件 | 即时反馈 |
|------|--------|------|------|----------|
| **拜访NPC** | 1AP | 关系+5~10，获得情报 | 需知道NPC位置 | 显示对话片段 |
| **送礼讨好** | 1AP + 物品 | 关系+15~25 | 需有合适礼物 | 显示NPC反应 |
| **帮助NPC** | 1-2AP | 关系+10，可能触发事件 | NPC有请求 | 显示帮助结果 |
| **请NPC帮忙** | 1AP | 使用NPC能力，消耗关系 | 关系≥40 | 显示NPC响应 |
| **参加聚会** | 2AP | 多人关系+5，情报 | 有聚会邀请 | 显示聚会场景 |

**四、事件参与行动**

| 行动 | AP消耗 | 效果 | 条件 | 即时反馈 |
|------|--------|------|------|----------|
| **处理事件** | 1-3AP | 根据事件类型 | 有待决事件 | 显示事件结果 |
| **发起行动** | 2-4AP | 创造新事件 | 需资源和技能 | 显示行动启动 |
| **调查线索** | 1AP | 获得隐藏信息 | 有线索/技能 | 显示发现的内容 |
| **参加会议** | 2AP | 影响决策，获得情报 | 有会议邀请 | 显示会议结果 |

### 2.3 行动解锁条件

不同技能等级解锁不同行动选项：

```
【谋略技能解锁的行动】

谋略L1：打听消息（1AP）── 搜集基础情报
谋略L2：解读密信（1AP）── 分析情报内容
谋略L3：建立情报网（3AP）── 长期情报来源
谋略L4：策划阴谋（4AP）── 发起政治事件
谋略L5：操控舆论（3AP）── 影响公众认知

【武力技能解锁的行动】

武力L1：街头格斗（1AP）── 小规模战斗
武力L2：护送任务（2AP）── 保护性战斗
武力L3：带队剿匪（3AP）── 中等规模战斗
武力L4：军事演练（2AP）── 提升部队能力
武力L5：发动战役（5AP）── 大规模军事行动

【经营技能解锁的行动】

经营L1：小买卖（1AP）── 基础交易
经营L2：开设店铺（3AP）── 建立固定收入来源
经营L3：组织商队（4AP）── 跨区域贸易
经营L4：产业扩张（5AP）── 大规模投资
经营L5：操控市场（4AP）── 影响区域物价
```

## 三、时间分配机制

### 3.1 行动点系统

**核心规则**：

```
每个游戏日，玩家拥有固定的行动点（AP）：

基础AP：每天 6 点
├── 可通过物品/状态增加：精力药水 +1AP
├── 可通过建筑增加：个人工坊 +1AP
└── 可通过技能减少消耗：谋略L3 行动消耗 -1AP

AP的约束意义：
├── 每天能做的事有限，需要规划
├── 不同行动消耗不同AP，体现"时间成本"
├── AP用完后当天不能再做主动行动
└── 未用的AP可以存起来（最多存+3，即每天最多9AP）

现实时间与AP的关系：
├── 1个游戏日 ≈ 现实2小时
├── 玩家可以每天登录，用完6AP
├── 或者存AP，周末一次用更多
└── 这适应碎片化和深度玩家两种节奏
```

**AP的叙事意义**：

```
AP不是抽象数字，而是"你的时间和精力"：

叙事呈现：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  "今天是第15日，你醒来时精力充沛。                             │
│   你有整整一天的时间来安排自己的活动。                         │
│   你估算了一下，大概能做5-6件事。                              │
│                                                              │
│   但昨晚铁壁将军的请求让你心神不宁...                          │
│   如果不去处理，可能错过机会。"                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘

这种叙事让AP成为"真实的时间感"，而不是冰冷的数字。
```

### 3.2 行动的时间成本

每个行动的AP消耗代表"这件事需要多久"：

| AP消耗 | 时间感觉 | 行动类型 |
|--------|----------|----------|
| **1AP** | 半天以内 | 快速拜访、送礼、打听消息 |
| **2AP** | 一整天 | 学习、工作、采集、社交聚会 |
| **3AP** | 需要准备 | 打怪、闭关修炼、调查 |
| **4AP** | 多日筹备 | 发起行动、组织商队、军事演练 |
| **5AP** | 重大投入 | 发动战役、产业扩张 |

### 3.3 每日节奏设计

```
玩家一天的典型节奏：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【早晨】登录，看到今天的状况                                  │
│     ├── 晨报：昨天发生了什么                                  │
│     ├── 待办：有哪些事情要做                                  │
│     ├── 状态：自己今天的能力和精力                            │
│     └── 计划：决定今天做什么                                  │
│                                                              │
│  【白天】执行行动，消耗AP                                      │
│     ├── 行动有即时反馈                                        │
│     ├── 每个行动都有叙事片段                                  │
│     ├── 行动后可能触发新事件                                  │
│     └── AP用完后可以查看结果                                  │
│                                                              │
│  【傍晚】结算今天的结果                                        │
│     ├── 资源变化总结                                          │
│     ├── 关系变化总结                                          │
│     ├── 技能进度总结                                          │
│     └── 新出现的待办事项                                      │
│                                                              │
│  【夜晚】世界后台运转                                          │
│     ├── NPC执行他们的行动                                     │
│     ├── 社会矛盾积累                                          │
│     ├── 事件按逻辑发展                                        │
│     └── 玩家离线后世界继续变化                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 四、行动成本与收益

### 4.1 成本类型

每个行动都有明确的成本和收益：

```
【成本类型】

一、行动点成本（时间）
   ├── 每个行动消耗固定AP
   ├── AP是稀缺资源，需要规划
   └── 某些行动需要更多AP（如发动战役需要5AP）

二、资源成本（金钱）
   ├── 送礼需要物品
   ├── 学习需要学费
   ├── 交易需要本金
   └── 发起行动需要启动资金

三、关系成本（人情）
   ├── 请NPC帮忙消耗关系值
   ├── 某些行动可能得罪其他NPC
   └── 帮助一方可能损害另一方的看法

四、风险成本（不确定性）
   ├── 打怪有受伤风险
   ├── 投资有失败风险
   ├── 某些行动可能触发负面事件
   └── 行动失败可能导致资源损失

五、机会成本（选择代价）
   ├── 做A就不能做B
   ├── 每天AP有限，选择有代价
   └── 不处理的事件可能恶化
```

### 4.2 收益类型

```
【收益类型】

一、资源收益（即时）
   ├── 打怪 → 金币+30~100
   ├── 采集 → 物品×3~5
   ├── 工作 → 金币+30~50
   └── 这些收益立刻显示，给玩家满足感

二、能力收益（长期）
   ├── 练习技能 → 技能经验+2~5
   ├── 学习技能 → 新技能解锁
   ├── 战斗 → 战斗经验+10
   └── 显示进度条增长，"我在变强"

三、关系收益（社交）
   ├── 拜访 → 关系+5~10
   ├── 送礼 → 关系+15~25
   ├── 帮助 → 关系+10
   └── 显示NPC的反应和态度变化

四、信息收益（情报）
   ├── 打听消息 → 获得线索
   ├── 调查 → 发现隐藏内容
   ├── 参加聚会 → 获得多方情报
   └── 这些信息可能解锁新事件或行动

五、叙事收益（体验）
   ├── 每个行动都有叙事片段
   ├── 行动后可能触发新故事
   ├── 系列行动可能形成故事线
   └── 这是游戏的核心价值
```

### 4.3 行动收益的具体设计

**即时反馈设计**：

```
每个行动完成后，立刻显示：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【行动完成】拜访铁匠老锤                                      │
│                                                              │
│  ───────────────────────────────────────────────────────    │
│                                                              │
│  你来到老锤的铁匠铺，他正在打磨一把剑。                        │
│                                                              │
│  "你来了？正好，我有些事想和你商量..."                         │
│                                                              │
│  【收益】                                                     │
│  ├─ 关系：老锤 +8 (从42 → 50)                                │
│  ├─ 情报：老锤透露新铁矿的消息                                │
│  ├─ 机会：老锤邀请你投资新铺                                  │
│  └─ 事件：触发"铁匠的投资邀请"事件                            │
│                                                              │
│  【消耗】                                                     │
│  ├─ AP：-1 (今日剩余：5AP)                                   │
│  └─ 时间：半天的拜访                                         │
│                                                              │
│  [接受邀请] [委婉拒绝] [询问详情]                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘

这就是即时反馈：
├── 立刻看到收益变化
├── 看到叙事片段
├── 可能触发新的选择
└── 知道自己消耗了什么
```

## 五、即时反馈系统

### 5.1 反馈的三层设计

```
【第一层：数值反馈】（立刻显示）

行动完成后，立刻显示数值变化：
├── 金币：+50 → 显示金币数变化
├── 关系：+10 → 显示关系进度条变化
├── 技能经验：+5 → 显示技能进度条变化
├── AP：-2 → 显示剩余AP

呈现方式：
┌─────────────────────────────────┐
│  ✓ 任务完成                      │
│                                  │
│  📊 收益：                        │
│     金币 +50 → 320               │
│     武力经验 +10 → 45/100        │
│                                  │
│  📉 消耗：                        │
│     AP -2 → 今日剩余 4           │
│                                  │
│  💬 老锤对你的好感上升了          │
└─────────────────────────────────┘

【第二层：叙事反馈】（行动的"故事"）

每个行动都有简短的叙事片段：
├── 拜访 → "你来到老锤的铁匠铺..."
├── 打怪 → "战斗激烈，你击退了强盗..."
├── 学习 → "导师传授了你新的技巧..."
├── 工作 → "一天的工作让你收获颇丰..."

这些叙事让行动不是"点一下按钮"，而是"经历了什么"

【第三层：后续反馈】（延迟显现）

某些行动有后续影响：
├── 投资 → 几天后收到收益报告
├── 帮助NPC → 几天后NPC可能回报
├── 做出的选择 → 后续事件反映其后果
├── 这些延迟反馈增加"选择有重量"的感觉

延迟反馈在晨报中呈现：
"几天前你帮助了老锤，今天他的新铺开业了，
他特意提到了你的帮助..."
```

### 5.2 反馈与驱动力的关系

```
即时反馈 → 玩家获得满足感 → 想再做一次 → 形成日常循环

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  玩家行动 → 看到效果 → 感到满足 → 想继续                      │
│                                                              │
│  示例：                                                       │
│                                                              │
│  玩家选择"练习武力"                                           │
│     ↓                                                        │
│  看到"武力经验+5，进度条从45变成50"                            │
│     ↓                                                        │
│  感到"我在变强，再练几次就能升级"                              │
│     ↓                                                        │
│  明天继续选择"练习武力"                                       │
│     ↓                                                        │
│  形成"我要把武力练到L3"的目标                                  │
│                                                              │
│  这就是即时反馈创造的核心驱动力！                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 进度可视化设计

```
【技能进度条】

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【武力】 L2 ████████░░░░░░░░ 45/100 → L3                   │
│                                                              │
│  今天练习武力后：                                             │
│  【武力】 L2 █████████░░░░░░░ 50/100 → L3                   │

## 六、核心驱动力设计

### 6.1 六大核心驱动

玩家每天登录游戏的六大核心驱动：

```
【驱动一：我要变强】

目标感：技能等级提升
├── 玩家看到技能进度条
├── 知道"再练几次就能升级"
├── 升级后会解锁新的行动和叙事
└── 这是最直接的驱动力

具体体现：
├── 每次练习都有经验增长反馈
├── 进度条可视化让目标清晰
├── 升级时有动画和叙事庆祝
└── 新等级解锁新选项，有期待感

【驱动二：我要积累资源】

获得感：金币和物品增加
├── 玩家需要金币来送礼、学习、投资
├── 每天打怪、工作、交易都能获得金币
├── 看到金币数字增长有满足感
└── 物品收集也有成就感

具体体现：
├── 打怪 → 金币+50，物品×2
├── 工作 → 金币+30
├── 交易 → 利润+20
├── 这些收益立刻显示，给即时满足

【驱动三：我要建立关系】

社交驱动：与NPC建立更深关系
├── 高关系NPC会主动帮忙
├── 高关系NPC会提供秘密情报
├── 高关系NPC会邀请参与事件
└── 关系是解锁内容的"钥匙"

具体体现：
├── 拜访 → 关系+8
├── 送礼 → 关系+15
├── 帮助 → 关系+10
├── 关系变化有叙事反馈："老锤对你的好感上升了"

【驱动四：我要处理事件】

紧迫驱动：不处理会恶化
├── 待决事件有时间限制（3天后过期）
├── 不处理可能导致关系下降
├── 不处理可能导致事件恶化
└── 这创造"必须登录"的压力

具体体现：
├── 待办提醒："铁壁将军的请求将在3天后过期"
├── 过期后果："将军对你的好感-10"
├── 恶化后果："边境局势进一步紧张"
└── 玩家感到"我必须处理"

【驱动五：我要探索未知】

好奇心驱动：发现新内容
├── 打听消息 → 可能发现新线索
├── 调查 → 可能发现隐藏内容
├── 拜访 → NPC可能透露新信息
└── 探索 → 可能触发新事件

具体体现：
├── 每次行动都可能获得新情报
├── 新情报显示为"线索卡片"
├── 线索可能指向新事件或行动
└── 玩家想知道"接下来会发生什么"

【驱动六：我要影响世界】

成就感驱动：自己的选择改变了世界
├── 参与事件 → 改变事件走向
├── 帮助NPC → 改变NPC命运
├── 做出决策 → 影响阵营关系
└── 这些变化在晨报中反映

具体体现：
├── 晨报："你帮助老锤后，他的新铺开业了"
├── NPC命运："你推荐的弟子成为了继承人"
├── 阵营变化："你的决策影响了边境谈判"
└── 玩家感到"我真的在影响这个世界"
```

### 6.2 驱动力的组合效应

```
单一驱动的局限：
├── 只想变强 → 玩家只练技能，忽略其他
├── 只想赚钱 → 玩家只打怪，忽略社交
├── 只想社交 → 玩家只拜访，忽略成长
└── 游戏体验单一化

驱动组合的设计：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  "我要把武力练到L3"                                          │
│     └── 需要练习（每天1AP）                                   │
│     └── 需要金币买装备（打怪赚钱）                             │
│     └── 需要导师指导（建立关系）                               │
│                                                              │
│  这迫使玩家组合多个行动：                                     │
│  ├─ 练习武力（变强）                                         │
│  ├─ 打怪赚钱（获得资源）                                      │
│  ├─ 拜访武师（建立关系）                                      │
│  └── 这些行动组合形成完整的日常循环                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 长期目标设计

```
除了日常驱动，玩家还需要长期目标：

【短期目标】（每天可见）
├── 技能升级（10天内可达）
├── 关系提升（5天内可达）
├── 事件处理（当天或本周）
└── 这些让每天有意义

【中期目标】（1-2周）
├── 加入某个团体
├── 解锁某个高级行动
├── 完成某个事件链
├── 达到某个社会阶层
└── 这些让每周有意义

【长期目标】（1个月+）
├── 成为某个领域的专家
├── 建立深厚的关系网络
├── 在某个阵营获得高位
├── 影响世界格局
└── 这些让整个赛季有意义

目标层次呈现：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【目标看板】                                                 │
│                                                              │
│  本周目标：                                                   │
│  ├─ 🎯 把武力提升到L3 (进度：50/100)                         │
│  ├─ 🎯 与铁壁将军达到"信任"关系 (进度：42→60)                 │
│  ├─ 🎯 加入铁匠工会 (需要：技能L2，关系≥40，金币≥50)           │
│                                                              │
│  本月目标：                                                   │
│  ├─ 🎯 成为边境联盟的"士绅"阶层                               │
│  ├─ 🎯 完成铁壁将军的事件链                                   │
│                                                              │
│  长期目标：                                                   │
│  ├─ 🎯 成为边境联盟的核心人物                                 │
│  └── 🎯 在边境建立自己的势力                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 七、与现有系统的整合

### 7.1 与事件系统的整合

```
行动系统不取代事件系统，而是补充它：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【事件来源】                                                 │
│                                                              │
│  世界生成事件（被动）                                         │
│  ├── 这些事件依然存在                                         │
│  ├── 玩家可以选择响应或不响应                                 │
│  └── 不响应可能导致恶化                                       │
│                                                              │
│  玩家主动行动（主动）                                         │
│  ├── 新增的行动系统                                          │
│  ├── 玩家每天主动做什么                                       │
│  └── 行动可能触发新事件                                       │
│                                                              │
│  【行动触发事件】                                             │
│                                                              │
│  拜访NPC → 可能触发NPC请求事件                                │
│  帮助NPC → 可能触发后续回报事件                               │
│  调查线索 → 可能触发新发现事件                                │
│  发起行动 → 直接创造新事件                                    │
│                                                              │
│  【事件需要行动处理】                                         │
│                                                              │
│  收到事件 → 需要分配AP来处理                                  │
│  复杂事件 → 需要多AP                                          │
│  系列事件 → 每个阶段需要行动推进                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 与技能系统的整合

```
技能不只是解锁叙事选项，也解锁行动选项：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【技能解锁行动】                                             │
│                                                              │
│  技能等级 → 解锁新的每日行动                                  │
│                                                              │
│  武力L1：街头格斗（1AP）                                      │
│  武力L2：护送任务（2AP）                                      │
│  武力L3：带队剿匪（3AP）                                      │
│  武力L4：军事演练（2AP）                                      │
│                                                              │
│  【技能减少成本】                                             │
│                                                              │
│  高技能等级 → 行动更高效                                      │
│                                                              │
│  武力L3：打怪收益+50%                                         │
│  武力L5：打怪AP消耗-1                                         │
│                                                              │
│  【技能决定成功概率】                                         │
│                                                              │
│  某些行动有成功概率                                           │
│  高技能 → 成功概率更高                                        │
│                                                              │
│  打怪成功率：                                                 │
│  ├── 武力L1：60%                                             │
│  ├── 武力L2：75%                                             │

│  └── 武力L4：95%                                             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 与NPC系统的整合

```
行动与NPC的关系：

┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  【拜访NPC】                                                  │
│                                                              │
│  消耗1AP，建立关系                                            │
│  高关系NPC → 拜访收益更多                                     │
│  特定NPC → 拜访可能获得特殊内容                               │
│                                                              │
│  【请NPC帮忙】                                                │
│                                                              │
│  消耗1AP + 关系值                                             │
│  使用NPC的能力（如让铁匠帮你打造装备）                         │
│  高关系NPC → 更愿意帮忙                                       │
│                                                              │
│  【帮助NPC】                                                  │
│                                                              │
│  消耗1-2AP                                                   │
│  可能触发NPC的回报                                            │
│  可能影响NPC的命运                                            │
│                                                              │
│  【NPC的日程】                                                │
│                                                              │
│  NPC有自己的日程，不是随时可拜访                              │
│  拜访需要在NPC的可拜访时间                                     │
│  NPC可能在特定地点才可拜访                                    │
│                                                              │
│  示例：                                                       │
│  老锤：白天在铁匠铺可拜访                                     │

│  秋实：朝会期间不可拜访                                       │

## 九、总结：核心玩法循环的七条原则

### 9.1 设计原则

```
原则一：玩家是演员，不是观众
├── 每天有明确的行动清单
├── 玩家主动选择做什么
└── 不是被动等待事件

原则二：时间是稀缺资源
├── AP系统约束每天能做的事
├── 不同行动消耗不同时间
└── 选择有代价，需要规划

原则三：行动有即时反馈
├── 每个行动立刻显示效果
├── 数值变化+叙事片段
└── 看不到效果就没有驱动力

原则四：成本与收益清晰
├── 每个行动明确显示消耗什么
├── 每个行动明确显示获得什么
└── 玩家可以理性规划

原则五：驱动来自目标
├── 短期目标（技能升级）
├── 中期目标（加入团体）
├── 长期目标（成为核心人物）
└── 目标让每天有意义

原则六：行动创造故事
├── 每个行动有叙事片段
├── 行动可能触发新事件
├── 系列行动形成故事线
└── 游戏体验是"经历"，不是"点击"

原则七：与现有系统无缝整合
├── 行动系统补充事件系统
├── 技能解锁行动选项
├── NPC是行动的对象和伙伴
└── 所有系统形成完整体验
```

### 9.2 设计验证清单

```
每个行动设计时，确认：

□ 这个行动消耗什么？（AP/金币/关系）
□ 这个行动获得什么？（数值/叙事/机会）
□ 这个行动有即时反馈吗？
□ 这个行动有叙事片段吗？
□ 这个行动为什么玩家想做？（驱动力）
□ 这个行动需要什么条件解锁？
□ 这个行动与其他系统如何关联？
```
