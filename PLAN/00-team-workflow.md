## 四、关键设计决策

### 1. 四级Agent架构

| 层级 | 模型等级 | 触发频率 | 主要职责 | 输出产物 |
|------|----------|----------|----------|----------|
| **世界级 (Chronos)** | 最高参数 (GLM-4/文心4.0) | 每日1次 | 世界平衡、历史推进 | 跨阵营重大事件、历史阶段推进 |
| **国家级** | 高参数 (GPT-4/Claude) | 每6小时 | 国家政策、外交决策 | 阵营内政策、外交行动 |
| **城邦级** | 中参数 (GPT-3.5/通义) | 每12小时 | 地方治理、资源调配 | 地方事件、资源分配 |
| **普通级** | 轻量模型 | 事件触发 | 日常行为、玩家交互 | NPC对话、个人事件 |

### 2. 叙事核心（Web优先）

| 设计点 | 说明 | 实现方式 |
|--------|------|----------|
| **每日小说** | 3000-5000字历史演义风格 | LLM生成 + 模板约束 |
| **"实时造英雄"** | 四国各1个主角NPC，实时生成传记 | Agent驱动 + 向量记忆 |
| **记忆管理** | 向量数据库 + 关键事件索引 | Pinecone/Milvus + 结构化存储 |
| **晨报系统** | 每日新闻汇编，玩家阅读入口 | 聚合Agent生成内容 |

### 3. 经济系统（MVP简化版）

| 要素 | 设计 | 说明 |
|------|------|------|
| **双资源** | 金币 + 影响力 | 金币=物质基础，影响力=社交资本 |
| **软上限** | 金币2000，影响力500 | 超限每日衰减，防止囤积 |
| **关系系统** | 7级关系（-100到+100） | 敌对→冷淡→陌生→认识→友好→亲密→至交 |
| **阶层系统** | 4级阶层 | 平民→士绅→贵族→王族 |
| **阵营声望** | -100到+100 | 死敌→敌对→不信任→中立→友好→尊敬→崇拜 |

### 4. 视觉风格

| 要素 | 设计 | 色值 |
|------|------|------|
| **风格** | 16-bit像素奇幻风 | - |
| **主色调** | 深空黑、羊皮纸、魔法白 | #1A1A2E、#F4E4C1、#F8F8FF |
| **阵营色** | 苍龙翡翠绿、霜狼冰蓝、金雀琥珀金、边境紫晶 | #2E7D32、#1976D2、#F57C00、#7B1FA2 |

### 5. 技能系统

| 要素 | 设计 | 说明 |
|------|------|------|
| **三条线** | 谋略、武力、经营 | 分别对应政治外交、战斗军事、商业工艺 |
| **4级等级** | 入门→精通→大师→传奇 | 1-3级、4-6级、7-9级、10级 |
| **核心理念** | 技能不是数值 | 是"能参与的故事类型"，技能等级决定可见叙事分支 |
| **MVP版** | 7个核心技能 | 每线2个（入门+精通）+ 1个通用 |

### 6. 事件系统

| 要素 | 设计 | 说明 |
|------|------|------|
| **事件类型** | 4大类×5个 = 20个模板 | 效忠选择、资源抉择、人际冲突、危机应对 |
| **触发机制** | 条件触发 + 随机触发 | 等级、资源、关系、阵营、技能、时间、状态 |
| **后果设计** | 资源/关系/声望/状态/事件链 | 多维度反馈，叙事包装 |
| **技能关联** | 技能等级决定选项 | 同一事件不同技能显示不同选项 |

## 七、评审检查清单

### 7.1 设计文档评审标准

#### 基础检查（所有文档必须通过）

| 检查项 | 说明 | 通过标准 |
|--------|------|----------|
| **目标明确** | 文档解决什么问题 | 阅读前3段能明确文档目标 |
| **背景清晰** | 为什么需要这个设计 | 引用相关前置文档 |
| **方案完整** | 包含输入、处理、输出 | 有具体实现方案，非概念描述 |
| **示例清晰** | 有具体的代码或流程示例 | 至少1个详细示例 |
| **可衡量** | 有明确的验收标准 | 能判断是否完成 |
| **关系清晰** | 与其他模块的关系 | 明确依赖和被依赖关系 |

#### 技术设计额外检查

| 检查项 | 说明 | 示例 |
|--------|------|------|
| **接口定义** | TypeScript接口完整 | `interface PlayerSkill { ... }` |
| **数据模型** | 数据库表结构或对象结构 | 字段、类型、约束 |
| **流程图** | 时序图或流程图 | ASCII图或文字描述 |
| **错误处理** | 异常情况处理方案 | 错误码、降级方案 |
| **性能考虑** | 性能指标和优化方案 | 响应时间、并发量 |

#### 美术设计额外检查

| 检查项 | 说明 | 示例 |
|--------|------|------|
| **参考图** | 风格参考、竞品参考 | 描述或链接 |
| **色值规范** | 具体的HEX/RGB值 | `#1A1A1A` |
| **尺寸规范** | 具体像素值 | 按钮：120×40px |
| **动画规范** | 时长、缓动函数 | 200ms ease-out |
| **响应式** | 不同屏幕尺寸适配 | 断点、布局变化 |

### 7.2 跨模块一致性检查

| 检查项 | 检查内容 | 检查方式 |
|--------|----------|----------|
| **术语一致** | 同一概念在不同文档中名称一致 | 全文搜索关键词 |
| **数值一致** | 同一数值在不同文档中相同 | 如：声望范围-100~+100 |
| **流程一致** | 同一流程在不同文档中描述一致 | 对比相关段落 |
| **依赖一致** | 模块A依赖模块B的描述一致 | 双向确认 |
| **风格一致** | 写作风格、格式规范一致 | 人工检查 |

### 7.3 评审流程

```
设计文档完成
     ↓
自我检查（按本清单）
     ↓
保存到指定路径
     ↓
发送评审请求给团队
     ↓
团队成员审查（2人+）
     ↓
   ├─ 通过 → 标记✅，合并入库
   └─ 不通过 → 反馈修改意见
         ↓
    修改后重新提交
```
