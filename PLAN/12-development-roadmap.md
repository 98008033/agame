# 开发路线

> **当前阶段：MVP（最小可行产品）**
>
> 目标：8周内上线可玩的叙事体验版本

---

## Phase 1: MVP（第1-8周）

### 核心目标

构建一个**叙事优先、Web运行、每日更新**的最小可用版本：
- 玩家每天登录阅读晨报
- 做出2-3个关键决策
- 看到决策的后果
- 世界持续运转

### 功能清单

| 模块 | 功能点 | 优先级 | 负责人 |
|------|--------|--------|--------|
| **基础架构** | 数据库设计 + API框架搭建 | P0 | |
| | 用户认证（简单JWT） | P0 | |
| **叙事引擎** | 晨报生成系统（LLM驱动） | P0 | |
| | 事件生成逻辑 | P0 | |
| | 叙事文本模板库（50个） | P0 | |
| **决策系统** | 二选一/三选一决策面板 | P0 | |
| | 决策后果计算 | P0 | |
| | 决策历史记录 | P0 | |
| **Agent系统** | Chronos主脑（每日运行） | P0 | |
| | 国家Agent简化版（3国共用） | P0 | |
| **Web前端** | 晨报阅读页（打字机效果） | P0 | |
| | 决策交互页 | P0 | |
| | 个人状态页 | P0 | |
| **内容** | 4个开局剧本（四国各1） | P0 | |
| | 20个通用事件模板 | P0 | |
| | 96个NPC基础设定导入 | P1 | |

### 技术选型

```
前端：React + TypeScript + Tailwind + Zustand
后端：Node.js + Express + PostgreSQL + Redis
LLM：Claude Sonnet（重要）+ GPT-3.5（日常）
部署：Vercel（前端）+ Railway（后端/数据库）
```

### 里程碑

| 周次 | 目标 | 交付物 |
|------|------|--------|
| **Week 1** | 基础架构 | 数据库 + API框架 + 认证 |
| **Week 2** | LLM接入 | LLM调用层 + 晨报生成Demo |
| **Week 3** | 核心逻辑 | Chronos + 事件系统 |
| **Week 4** | 前端框架 | 页面结构 + 数据流 |
| **Week 5** | 决策系统 | 决策流程 + 后果计算 |
| **Week 6** | 内容填充 | 20个事件 + 4个开局 |
| **Week 7** | 集成测试 | 端到端流程 + Bug修复 |
| **Week 8** | 上线准备 | 性能优化 + 文档 + 部署 |

### 成功标准

- [ ] 玩家可以注册/登录
- [ ] 每天看到不同的晨报
- [ ] 能做出选择并看到即时后果
- [ ] 世界状态持续保存
- [ ] 单个事件链能完整体验（至少3步）
- [ ] 页面加载 < 3秒
- [ ] LLM响应 < 5秒

## Phase 2: 核心体验（第9-16周）

### 目标

增强叙事深度，增加系统复杂度，提升留存。

### 功能清单

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **叙事** | 事件链系统（多章节连续叙事） | P0 |
| | 人物传记系统 | P1 |
| | 历史编年史（可回顾） | P1 |
| **Agent** | 独立国家Agent（3国分开） | P0 |
| | NPC Agent对话系统 | P1 |
| **系统** | 效忠/背叛流程完整版 | P0 |
| | 声望系统（NPC关系网） | P1 |
| | 资源系统（金币/影响力） | P1 |
| **内容** | 事件模板扩展到50个 | P1 |
| | 新增10个独特NPC | P2 |

### 成功标准

- 7日留存率 > 30%
- 平均每日会话时长 > 8分钟
- 叙事质量评分 > 4/5

## 功能优先级说明

```
P0 = MVP必须实现，否则无法上线
P1 = 上线后4周内必须实现，影响核心体验
P2 = 上线后8周内实现，增强长期留存
```

---

## 风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| LLM成本超支 | 中 | 高 | 实现缓存，限制调用频率 |
| 开发周期超出 | 高 | 中 | 严格P0边界，果断砍P1/P2 |
| 叙事质量不稳定 | 中 | 高 | 建立模板库，人工审核 |
| 玩家留存低 | 中 | 高 | 快速迭代，数据驱动优化 |

---

## 决策原则

**任何功能加入前，必须回答**：

1. 这个功能是叙事必需的吗？
2. 没有这个功能，核心体验还能成立吗？
3. 这个功能会增加LLM调用成本吗？
4. 这个功能能在一个迭代周期内完成吗？

**如果第2个答案是"否"，或者第4个答案是"否"，延后实现。**
