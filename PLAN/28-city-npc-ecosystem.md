# Agame 城市NPC生态系统

> **核心理念**：每个城市都是一个独立运转的生态，拥有支撑城市基本功能的NPC。这些NPC大部分是Template类型，通过技能系统提供服务，与玩家进行日常交互。
>
> **文案风格**：NPC不只是功能性的"NPC名"，而是有故事、有性格的真实人物。每个NPC都应该有自己的小故事和独特气质。

## 六、使用说明

### 6.1 NPC生成原则

1. **必要性优先**：每个NPC都必须有其存在的功能价值
2. **生态完整**：每个城市都有支撑其运转的基本职能NPC
3. **层次清晰**：Agent负责决策和剧情，Template提供服务
4. **阵营特色**：不同阵营的同类NPC有不同的风格和能力

### 6.2 动态生成

对于中小城市，可以使用模板动态生成：

```typescript
interface CityNPCGenerator {
  // 基础配置
  cityType: 'capital' | 'major' | 'minor' | 'village';
  population: number;
  faction: string;

  // 必需NPC
  requiredNPCs: {
    governance: number;  // 管理类
    military: number;    // 军事类
    commerce: number;    // 商业类
    craft: number;       // 工艺类
    service: number;     // 服务类
  };

  // 生成规则
  generate(): TemplateNPC[];
}
```

### 6.3 与现有系统对接

- 这些Template NPC可以直接使用[27-template-npc-system.md](27-template-npc-system.md)中的技能系统
- Agent NPC可以升级到完整的Agent系统
- 所有NPC都可以参与事件链和晨报生成
