# Agame 经济与社会系统设计（MVP极简版）

> **设计原则**：严格遵循MVP极简原则，只保留支撑叙事体验的最小经济系统。延后复杂经济机制，优先上线验证。

---

## 一、MVP经济系统（双资源）

### 1.1 资源类型

根据 `15-priority-web-narrative.md`，MVP阶段**仅保留两种资源**：

| 资源 | 作用 | 叙事意义 |
|------|------|----------|
| **金币** | 通用货币，用于交易、建设、招募 | 代表经济实力与物质基础 |
| **影响力** | 社交资本，用于外交、开启事件 | 代表声望与人脉网络 |

**延后资源**：情报（P1阶段加入）、资源点数（P2阶段，替代复杂领地系统）

---

### 1.2 资源获取途径

#### 金币获取

| 来源 | 公式 | 冷却/限制 |
|------|------|----------|
| **日常产出** | 基础值 50/游戏日 | 需有领地或效忠阵营 |
| **任务奖励** | 任务难度 × 10-100 | 完成任务后一次性 |
| **事件奖励** | 随机 20-200 | 特定事件选择 |
| **登录奖励** | 连续登录天数 × 10 | 每日首次登录 |

**MVP简化说明**：
- 移除复杂贸易系统（P2阶段再加入）
- 移除领地建设产出（简化为"有/无领地"状态）
- 移除NPC交易（简化为事件奖励）

#### 影响力获取

| 来源 | 公式 | 限制条件 |
|------|------|----------|
| **完成事件** | 事件重要性 × 5-20 | 成功处理事件后 |
| **帮助NPC** | 10-30/次 | 完成NPC请求 |
| **阵营贡献** | 效忠阵营后 ×1.5倍 | 需有效忠关系 |
| **晨报传播** | 特殊新闻相关 +5-15 | 随机触发 |

---

### 1.3 资源消耗

#### 金币消耗

| 用途 | 消耗 | 说明 |
|------|------|------|
| **建设领地** | 200-500/次 | MVP简化为"升级"概念 |
| **招募NPC** | 50-200/人 | 解锁特定事件选择 |
| **贿赂/打点** | 30-100/次 | 某些事件的中间选项 |
| **购买情报** | 100/条 | P1阶段开放 |

#### 影响力消耗

| 用途 | 消耗 | 说明 |
|------|------|------|
| **开启事件** | 10-50/次 | 主动触发的特殊事件 |
| **外交请求** | 20-80/次 | 向NPC提出帮助请求 |
| **跳过等待** | 30/次 | 加速某些事件冷却 |
| **声望兑换** | 50/级 | 提升阵营声望 |

---

### 1.4 资源平衡公式

#### 日常收支平衡（设计目标）

```
目标：轻度玩家每日净增长 +20~50
      活跃玩家每日净增长 +50~100

每日收入（基准）：
  金币：50（基础）+ 30（任务）+ 20（事件）= 100
  影响力：10（事件）+ 10（社交）= 20

每日消耗（基准）：
  金币：30（日常）+ 40（建设）= 70
  影响力：5（社交）+ 10（事件）= 15

净增长：
  金币：+30
  影响力：+5
```

#### 资源上限与衰减

| 资源 | 软上限 | 超限衰减 |
|------|--------|----------|
| 金币 | 2000 | 每日 -1%（通胀模拟） |
| 影响力 | 500 | 每日 -2%（声望维护成本） |

**设计理由**：
- 防止囤积，鼓励消费
- 模拟真实经济动态
- 保持玩家持续参与的动力

---

```
设计原则：资源变化必须伴随叙事反馈

金币增加：
  ✓ "你的领地今日上缴了税金 50金币"

  ✓ "你的仓库扩建完成，花费 300金币"

  ✓ "你动用关系打探消息，影响力 -30"
  ✓ "你的请求让老友为难，影响力 -20"
```

## 二、社会系统

### 2.1 关系网络（NPC关系）

#### 关系等级

```
数值范围：-100 ~ +100

-100 ~ -50：敌对（会主动阻碍玩家）
-49 ~ -20：冷淡（不愿帮助）
-19 ~ +20：陌生（基础对话）
+21 ~ +40：认识（愿意交易）
+41 ~ +60：友好（会提供帮助）
+61 ~ +80：亲密（可请求重要协助）
+81 ~ +100：至交（生死与共）
```

#### 关系变化规则

| 行为 | 影响 | 备注 |
|------|------|------|
| **完成NPC请求** | +10 ~ +25 | 根据请求难度 |
| **帮助NPC解决危机** | +20 ~ +40 | 重大事件 |
| **日常互动** | +2/次 | 每日限1次 |
| **拒绝请求** | -5 ~ -15 | 根据请求重要性 |
| **长时间不互动** | -3/游戏周 | 自然衰减 |
| **背叛行为** | -50 ~ -100 | 可能转为敌对 |

#### MVP简化说明

- **关系数量限制**：MVP阶段每位玩家最多维护 20 个重要NPC关系
- **被动关系**：非重要NPC关系为"被动"状态，只记录数值，无主动事件
- **关系冷却**：同一NPC每日互动次数有限（防止刷好感）

---

### 2.2 社会阶层

#### 阶层体系

```
平民 → 士绅 → 贵族 → 王族
(1-10级) (11-25级) (26-40级) (41级+)
```

| 阶层 | 等级 | 特权 | 可参与事件 |
|------|------|------|----------|
| **平民** | 1-10 | 基础交易 | 村庄级事件 |
| **士绅** | 11-25 | 领地建设、任务发布 | 城镇级事件 |
| **贵族** | 26-40 | 外交谈判、阵营决策 | 国家级事件 |
| **王族** | 41+ | 战争决策、历史走向 | 世界级事件 |

#### 晋升路径

**平民 → 士绅**：
- 完成 5 个村庄级事件
- 获得任意NPC友好度 ≥40
- 金币 ≥300

**士绅 → 贵族**：
- 完成 1 个阵营效忠任务
- 影响力 ≥100
- 等级 ≥26

**贵族 → 王族**：
- 参与 1 次国家级重大决策
- 在至少 2 个阵营中有 ≥60 声望
- 等级 ≥41

---

### 2.3 阵营声望

根据 `04-character-system.md`，阵营声望范围为 **-100 ~ +100**

| 声望区间 | 称号 | 效果 |
|----------|------|------|
| -100 ~ -80 | 死敌 | 无法进入该阵营领地，被通缉 |
| -79 ~ -50 | 敌对 | 会被拒绝交易，任务受限 |
| -49 ~ -20 | 不信任 | 交易价格 +50%，任务奖励 -30% |
| -19 ~ +20 | 中立 | 正常交易 |
| +21 ~ +50 | 友好 | 任务奖励 +20%，优先获得任务 |
| +51 ~ +80 | 尊敬 | 可购买特殊物品，优先情报 |
| +81 ~ +100 | 崇拜 | 可参与高层决策，特殊事件邀请 |

#### 声望获取

| 行为 | 声望变化 |
|------|----------|
| 完成阵营任务 | +5 ~ +20 |
| 效忠阵营 | +30（一次性） |
| 帮助阵营NPC | +3 ~ +10 |
| 攻击阵营成员 | -10 ~ -50 |
| 背叛阵营 | -80（一次性） |
| 传播阵营负面消息 | -5 ~ -15 |

## 三、数据模型

### 3.1 玩家经济状态

```typescript
interface PlayerEconomy {
  // 资源
  gold: number;           // 金币，软上限 2000
  influence: number;      // 影响力，软上限 500

  // 收入记录（用于衰减计算）
  dailyIncome: {
    gold: number;
    influence: number;
    date: string;         // 游戏日
  }[];

  // 消费记录
  dailyExpense: {
    gold: number;
    influence: number;
    date: string;
  }[];
}
```

### 3.2 关系数据

```typescript
interface Relationship {
  npcId: string;          // NPC唯一ID
  level: number;          // -100 ~ +100
  history: Event[];       // 互动历史
  lastInteract: string;   // 最后互动时间
  isImportant: boolean;   // 是否重要NPC（MVP限制20个）
}

interface SocialStatus {
  tier: 'civilian' | 'gentleman' | 'noble' | 'royal';
  tierLevel: number;      // 1-50
  factionRep: {
    canglong: number;     // 苍龙帝国声望
    shuanglang: number;   // 霜狼联邦声望
    jinque: number;       // 金雀花王国声望
    border: number;       // 边境联盟声望
  };
  relationships: Relationship[];
}
```

### 3.3 每日结算流程

```
1. 资源产出结算
   - 计算基础收入
   - 应用领地产出倍率（如有领地）
   - 应用阵营效忠倍率（如有效忠）

2. 资源衰减结算
   - 超过软上限的部分按百分比衰减
   - 金币：-1%（超过2000部分）
   - 影响力：-2%（超过500部分）

3. 关系衰减结算
   - 检查所有关系
   - 超过7游戏日未互动：-3
   - 生成"关系提醒"事件（如果降至阈值）

4. 生成晨报条目
   - 资源变化转化为叙事文本
   - 关系变化转化为社交新闻
```

## 四、与Agent系统整合

### 4.1 Chronos经济干预

```
Chronos每日运行时检查：

1. 经济平衡检查
   IF 某玩家金币 > 3000:
     → 触发"财富引来觊觎"事件
     → 选项：投资/藏匿/挥霍

   IF 某玩家影响力 > 600:
     → 触发"名声带来的负担"事件
     → 选项：利用/隐藏/传播

2. 关系平衡检查
   IF 某NPC关系 > 80 且 该NPC是某阵营高层:
     → 触发"阵营猜忌"事件
     → 选择：解释/利用/无视

3. 阶层晋升检查
   IF 玩家满足晋升条件:
     → 生成晋升事件
     → LLM生成叙事描述
```

### 4.2 国家Agent经济决策

```
国家Agent每6小时运行时：

1. 评估经济策略
   - 高金币储备 → 可能发起建设项目
   - 高影响力 → 可能发起外交行动
   - 资源短缺 → 可能向玩家征税/求助

2. 生成经济相关事件
   - "XX国财政紧张，寻求投资"（玩家可投资换取声望）
   - "XX国贸易代表团来访"（玩家可参与贸易获利）
```

## 六、设计验证问题

**每个经济/社交功能加入前，问自己：**

1. 这个功能能让故事更精彩吗？
2. 没有这个功能，玩家还能理解自己的社会地位吗？
3. 这个系统会增加玩家的认知负担吗？
4. 这个计算需要实时进行吗？（能否离线后批量结算）

**如果第2个答案是"否"，或第3个答案是"是"，考虑简化或延后。**
