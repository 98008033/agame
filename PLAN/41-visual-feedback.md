# Agame 可见反馈系统设计

> **核心问题**：玩家能看到变化吗？复杂的社会系统如何变得可见可感？
> **设计目标**：让后台运转的复杂社会系统通过多层次的反馈呈现给玩家，让选择有意义，让变化可感知。

---

## 一、反馈系统总览

### 1.1 核心设计理念

```
反馈的本质：不是"显示数值"，而是"讲述变化"

传统游戏反馈：
  "关系+10" → 冰冷的数值变化

Agame反馈设计：
  "老锤看到你帮村民修理农具，眼神中多了几分信任"
  → 叙事化的变化，玩家感受到人与人之间的连接

反馈三原则：
1. 叙事优先：变化用故事呈现，而非纯数值
2. 层次分明：即时反馈抓眼球，长期反馈留印象
3. 选择有响：玩家行为必须产生可见的后果
```

### 2.2 数值变化提示

数值变化不是冷冰冰的"+10"，而是带有叙事意味的呈现：

**资源变化动画**

```
金币变化：
├── 增加：金币图标微弹，数字滚动增长
│   叙事："你收到了..."
├── 减少：金币图标轻微下沉，数字滚动减少
│   叙事："你支付了..."

影响力变化：
├── 增加：星星图标闪烁，数值平滑上升
│   叙事："你的影响力扩大了..."
├── 减少：星星黯淡，数值平滑下降
│   叙事："你的决定让一些人失望..."

阵营声望变化：
├── 增加：阵营徽章高亮，边框发光
│   叙事："XX阵营对你更信任了"
├── 减少：阵营徽章黯淡，边框变灰
│   叙事："XX阵营对你产生了疑虑"
```

**变化提示样式规范**

```typescript
interface ChangeNotification {
  // 显示位置
  position: 'inline' | 'floating' | 'modal';

  // 动画时长
  duration: {
    appear: 300;      // 出现动画
    display: 2000;    // 显示时间
    fadeOut: 500;     // 消失动画
  };

  // 视觉层次
  priority: {
    high: 'modal';    // 高优先级（重大变化）用弹窗
    medium: 'inline'; // 中优先级（资源变化）用内嵌
    low: 'floating';  // 低优先级（微小变化）用悬浮
  };

  // 叙事化文本
  narrative: {
    template: string; // 叙事模板
    context: string;  // 具体情境描述
  };
}
```

### 2.3 关系线变化动画

关系是Agame的核心系统，关系变化必须有视觉呈现：

**关系网络动画**

```
关系变化触发动画：

关系上升（+10~+30）：
├── 连线变粗（粗细对应关系强度）
├── 连线颜色渐变（从灰→绿→蓝→金）
├── NPC头像框发光（脉冲效果）
└── 悬浮文字："XX对你的信任增加"

关系下降（-10~-30）：
├── 连线变细
├── 连线颜色渐变（从绿→灰→橙→红）
├── NPC头像框变暗
└── 悬浮文字："XX对你的信任减少"

重大关系变化（±30+）：
├── 连线剧烈变化动画（1秒）
├── NPC头像缩放动画
├── 特殊音效
└── 弹窗叙事："你和XX的关系发生了重大变化..."
```

**关系线颜色编码**

```
关系值区间 → 连线颜色 → 视觉含义

-100~-60: 红色深红 → 血仇/死敌
-60~-30:  橙色     → 敌对/厌恶
-30~0:    灰色     → 疏远/陌生
0~30:     浅绿     → 认识/浅交
30~60:    绿色     → 友好/合作
60~80:    蓝色     → 深交/信任
80~100:   金色     → 生死之交
```

### 2.4 事件触发通知

新事件触发时，需要通知玩家但不打断体验：

**通知层级**

```
紧急事件（需要立即响应）：
├── 屏幕中央弹窗
├── 特殊音效
├── NPC头像闪烁
└── "紧急"标签

重要事件（当日需要处理）：
├── 右上角通知卡
├── 轻微提示音
├── 事件图标
└── "重要"标签

普通事件（可延后处理）：
├── 底部消息栏
├── 无音效
└── 图标+简要描述

背景事件（不需处理，只需知晓）：
├── 晨报提及
├── 不触发单独通知
└── 归入"昨日要闻"
```

**通知UI设计**

## 五、反馈节奏与信息管理

### 5.1 信息层级划分

不是所有信息都需要通知玩家，需要分层处理：

```
信息层级划分：

第一层：必须通知（打断体验）
├── 紧急事件（需要立即处理）
├── 重大人生事件（NPC死亡/继承）
├── 关系剧烈变化（±30+）
├── 阵营效忠/背叛
└── 玩家死亡/继承

第二层：应该通知（不打断但提示）
├── 新事件触发（当日需处理）
├── 关系明显变化（±15~30）
├── 技能等级提升
├── NPC职位变化（重要NPC）
├── 社会态势变化（达到警示阈值）
└── 玩家声望变化

第三层：可以通知（汇总呈现）
├── 关系小幅变化（±10以下）
├── NPC日常动态
├── 经济微小波动
├── 背景事件（不需处理）
└── 玩家行为的微小后果

第四层：不需通知（后台运转）
├── NPC内部关系变化（不影响玩家）
├── 社会隐性矛盾积累
├── 资源供需微调
├── Template NPC日常行为
└── 后台数值更新
```

### 5.2 反馈频率控制

控制反馈频率，避免信息过载：

```
反馈频率限制：

即时反馈：
├── 每个选择必须有反馈（不限制）
├── 但同一事件的多个后果可合并展示
└── 最多同时显示3个变化提示

事件通知：
├── 紧急事件：不限制
├── 重要事件：每日最多3个
├── 普通事件：每日最多5个
└── 背景事件：汇总在晨报

晨报内容：
├── 世界风云：每阵营最多1条
├── 身边变化：最多5条
├── 昨日回顾：最多1条详细
├── 今日预兆：最多3条
└── 总字数控制在500字内

状态面板：
├── 默认显示摘要
├── 点击展开详情
├── 自动更新但不弹窗
└── 变化超过阈值才提示

长期反馈：
├── NPC轨迹：每NPC每10日最多1条新增
├── 社会格局：每20日更新一次
├── 个人传记：重要事件才记录
└── 历史时间线：重大事件才记录
```

### 5.3 信息聚合策略

多个信息聚合呈现，减少干扰：

```
信息聚合策略：

事件后果聚合：

  // 动画规则
  animations: {
    changeThreshold: 5;   // 变化超过5才动画
    highlightDuration: 2000;
    fadeOutDelay: 3000;
  };
}

```

   - [ ] 是否正确归类信息层级？
   - [ ] 是否符合频率限制？

   - [ ] 通知是否过于频繁？
   - [ ] 是否可以延后呈现？

   - [ ] 动画性能是否可接受？
