# 战斗与冲突系统

## 九、战斗与冲突系统

### 9.1 设计原则

- **非实时**，文字描述为主
- **策略性**体现在准备阶段而非操作阶段
- **结果**由多因素决定（属性+装备+魔法+士气+地形+随机）

### 9.2 冲突类型

**个人战斗**（冒险者路径）：
```
准备：选择装备 + 魔法 + 战术倾向
执行：系统根据多因素自动生成结果
呈现：文字描述战斗过程
结果：胜利/失败 → 经验/物品/声望变化
```

**领地战争**（领主路径）：
```
准备：调配军队 + 选择将领 + 制定战略
执行：按游戏日推进（每天生成当日战报）
呈现：每天生成战报，玩家可在战报间调整战略
结果：领土变化 + 资源消耗 + 士气变化
```

**外交冲突**（外交家路径）：
```
手段：信息、承诺、威胁、贿赂
对手：NPC的性格、利益、关系网
结果：关系变化 + 资源变化 + 新事件
```
