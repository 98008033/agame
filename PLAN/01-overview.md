# 核心概念

## 一、游戏定位

### 1.1 一句话描述

在一个由AI自主运转的乱世中，你选择效忠、背叛或自立，成为推动历史的那颗石子——或者被历史碾碎。

### 1.2 核心卖点

| 卖点 | 描述 | 玩家价值 |
|------|------|----------|
| **每日网文小说** | 抽取游戏内信息，单独输出小说。四国主角的命运在网文连载中展开 | 低门槛的日常内容消费 |
| **核心游戏体验** | 游戏本身提供策略决策和Agent交互，小说是游戏的副产品 | 真正的游戏乐趣 |
| **你的选择改变小说** | 玩家的每个决策都会被记录进小说，真正成为故事的一部分 | 参与感与成就感 |
| **世界持续运转** | 世界在玩家不在线时依然运转——战争爆发、王朝更替、瘟疫蔓延 | 真实感与紧迫感 |
| **四个阵营选择** | 你可以选择效忠任何一国，也可以选择成为边境之王 | 自由度与重玩价值 |
| **真正的蝴蝶效应** | 小村庄的选择可能改变三国格局 | 策略深度 |
| **隐藏的上帝之手** | 系统AI在幕后维持世界平衡，玩家只能感受到"时势" | 神秘感与探索乐趣 |

### 1.3 体验节奏

```
轻度入口 → 深度内核 → 永久后果

日常：每日登录10-20分钟
  ├─ 完成日常活动（打怪、采集、学习、制造）
  ├─ 处理2-3个关键决策
  └─ 阅读晨报（可选，游戏的副产品）

深入：每周1-2次30-60分钟深度体验
  ├─ 副本探险与BOSS挑战
  ├─ 技能学习与升级规划
  ├─ 重大抉择与事件链
  ├─ NPC深度交互与关系经营
  └─ 制造装备与资源管理

极端：重大事件期间（战争/政变/瘟疫）
  └─ 需要密集决策，可能改变角色命运
```

**核心机制**：

| 机制 | 说明 | 体验影响 |
|------|------|----------|
| **永久死亡** | 玩家角色可能因决策失误、危机事件或战斗失败而死亡，无法复活 | 增加决策重量感，每个选择都有真实风险 |
| **角色成长** | 通过打怪、采集、学习、制造来提升角色能力，像NPC一样生活 | 提供RPG式的成长感和成就感 |
| **小说副产品** | 每日生成的网文小说记录世界动态，是游戏过程的叙事呈现 | 可选阅读，不是核心玩法，而是世界观补充 |

**关于角色死亡**：
- 角色死亡是永久性的，但玩家可以继承遗产（资源、关系、部分影响力）开始新角色
- 死亡不是终点，而是故事的一部分——你的前角色会成为世界历史中的NPC
- 新角色可以选择继承前角色的遗志，或走完全不同的道路

## 二、目标用户

### 2.1 核心用户画像

**主要目标群体：18-35岁游戏爱好者**

| 特征 | 描述 |
|------|------|
| **年龄** | 18-35岁，包括学生和年轻职场人士 |
| **游戏偏好** | RPG、模拟经营、策略游戏，喜欢角色成长和开放世界体验 |
| **时间特点** | 碎片化时间为主，但也需要可沉浸的深度内容 |
| **核心诉求** | 想体验"活着的世界"，通过打怪、种田、升级来发展角色 |
| **付费意愿** | 愿意为角色成长、时间节省和外观定制付费 |
| **平台习惯** | Web/移动端，偏好即开即玩，进度可跨设备同步 |

**典型用户画像**：
- **小王（22岁，大学生）**：课余时间打怪升级、采集资源，喜欢角色成长感
- **小李（28岁，产品经理）**：下班后完成日常任务，周末深入副本探险
- **小张（32岁，自由职业）**：时间灵活，追求角色强度和收集成就

### 2.2 用户需求分析

| 需求层级 | 需求描述 | 游戏满足方式 |
|----------|----------|--------------|
| **基础层** | 娱乐消遣、打发时间 | 每日任务、挂机收益、快速战斗 |
| **成长层** | 角色变强的爽感 | 打怪升级、技能提升、装备打造 |
| **情感层** | 成为故事一部分 | 决策影响世界、被写入小说 |
| **社交层** | 分享独特经历 | 个人传记可分享、排行榜 |
| **成就层** | 证明智慧与策略 | 多结局、称号系统、排行榜 |

### 2.3 用户场景

**场景一：日常成长（日常）**
> 小王每天登录游戏，先去野外打怪获取材料，然后到铁匠铺制作装备，最后处理几个事件决策。看着角色一天天变强，很有成就感。

**场景二：周末探险（深入）**
> 周六下午，小李花1小时深入副本探险，击败BOSS获得稀有材料，用来学习新技能。之后参与一个事件链，决定向哪方势力告发边境走私案。

**场景三：深夜沉浸（深度）**
> 周五晚上，小王难得有空，花1小时处理一个复杂事件链。从边境走私案开始，逐步揭开跨国阴谋，最后需要在多个势力间做出关键抉择。这种"沉浸式"体验是他每周最期待的时刻。

## 三、核心玩法要素

### 3.1 玩法循环

玩家像NPC一样生活在一个自主运转的世界中，通过日常行动积累能力，通过关键决策影响命运。

```
┌─────────────────────────────────────────────────────────────┐
│                     核心玩法循环                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │ 日常行动 │ → │ 能力提升 │ → │ 关键决策 │             │
│   │ 种田/打怪│    │ 技能/装备│    │ 影响世界 │             │
│   └──────────┘    └──────────┘    └────┬─────┘             │
│        ↑                               │                    │
│        └───────────────────────────────┘                    │
│                                                             │
│   日常行动类型：                                             │
│   ├─ 战斗：打怪、完成任务、参与战争                          │
│   ├─ 生产：种田、采矿、锻造、炼金                            │
│   ├─ 学习：研究技能、学习魔法、读书                          │
│   └─ 社交：交易、建立关系、情报收集                          │
│                                                             │
│   世界在后台持续运转（Agent系统）                              │
│   玩家是活生生的居民，不是世界的中心                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**玩家日常体验**：

| 行动类型 | 示例 | 产出 |
|----------|------|------|
| **战斗** | 剿灭土匪、护送商队、参与战役 | 装备、金钱、声望、战斗经验 |
| **生产** | 耕种农田、开采矿石、制作物品 | 资源、商品、经济收入 |
| **学习** | 研究技能书、向NPC请教、实践锻炼 | 技能点、新能力、知识 |
| **交易** | 买卖商品、投资产业、签订契约 | 金钱、物资、商业关系 |

### 3.3 玩家成长路径

```
新手期（第1-7天）
├── 了解世界观和阵营
├── 掌握基础行动（战斗/生产/学习/交易）
├── 体验第一个关键决策
└── 建立初始关系网

成长期（第2-4周）
├── 深入事件链
├── 建立声望和影响力
├── 参与阵营事务
└── 发展专业技能路线

成熟期（第1个月+）
├── 影响世界格局
├── 培养势力/继承人
├── 挑战高难度内容
└── 追求独特成就与称号
```

## 四、设计原则

### 4.1 核心设计哲学

> **"玩家是历史的一部分，不是历史的中心"**

**原则一：叙事优先**
- 任何功能加入前，先问：这个功能能让故事更精彩吗？
- 如果砍掉这个功能，核心体验还能成立吗？

**原则二：AI驱动但可控**
- 使用AI生成内容，但通过模板和约束保证质量
- 关键节点人工设计，填充内容由AI生成

**原则三：低门槛高深度**
- 日常体验简单：打怪、种田、做选择
- 深度体验丰富：装备打造、技能搭配、策略规划

**原则四：世界自主运转**
- 没有玩家，世界也会自己运转
- 玩家是变量，不是必需品

### 4.2 设计检查清单

```
任何功能设计时，确认以下问题：

□ 这个功能是叙事必需的吗？
□ 这个功能能在Web上良好运行吗？
□ 这个功能需要复杂的UI吗？
□ 这个功能会增加LLM调用成本吗？
□ 如果砍掉这个功能，核心体验还能成立吗？

如果后三个问题中有超过一个"是"，考虑简化或延后。
```

## 五、项目边界

### 5.1 不做的事情

| 范围 | 说明 | 原因 |
|------|------|------|
| **不做3D/复杂视觉** | Web优先，文字为主 | 控制成本，聚焦叙事 |
| **不做实时PvP** | 异步交互为主 | 降低技术复杂度 |
| **不做复杂战斗系统** | 战斗简化为决策结果 | 非核心体验 |
| **不做阵营隔离** | 四国玩家在同一世界 | 技术可行，增强世界真实感 |
| **不做付费赢** | 付费仅用于装饰/便捷 | 保证公平性 |

### 5.2 MVP范围

**必须实现（P0）**：
- 角色成长系统（打怪、采集、学习、升级）
- 每日晨报生成与阅读
- 基础决策系统
- 世界状态保存
- 4个开局剧本

**延后实现（P1/P2）**：
- 完整技能树
- 领地建设
- 排行榜系统
- 微信小游戏
