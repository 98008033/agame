# 核心概念

## 一、游戏定位

### 1.1 一句话描述

在一个由AI自主运转的乱世中，你选择效忠、背叛或自立，成为推动历史的那颗石子——或者被历史碾碎。

### 1.2 核心卖点

| 卖点 | 描述 | 玩家价值 |
|------|------|----------|
| **每日网文小说体验** | 每天阅读3000-5000字连载小说，四国主角的命运在你面前展开 | 低门槛的日常内容消费 |
| **你的选择改变小说** | 玩家的每个决策都会被记录进小说，真正成为故事的一部分 | 参与感与成就感 |
| **世界持续运转** | 世界在玩家不在线时依然运转——战争爆发、王朝更替、瘟疫蔓延 | 真实感与紧迫感 |
| **四个阵营选择** | 你可以选择效忠任何一国，也可以选择成为边境之王 | 自由度与重玩价值 |
| **真正的蝴蝶效应** | 小村庄的选择可能改变三国格局 | 策略深度 |
| **隐藏的上帝之手** | 系统AI在幕后维持世界平衡，玩家只能感受到"时势" | 神秘感与探索乐趣 |

### 1.3 体验节奏

```
轻度入口 → 深度内核

日常：每日登录5-10分钟，阅读世界晨报，做2-3个关键决策
深入：每周1-2次30分钟深度体验——外交谈判、重大抉择、事件链
极端：重大事件期间（战争/政变/瘟疫）需要密集决策
赛季：每现实月为一个"赛季"，有赛季目标和排行，驱动长线留存
```

## 二、目标用户

### 2.1 核心用户画像

**主要目标群体：25-40岁策略游戏爱好者**

| 特征 | 描述 |
|------|------|
| **年龄** | 25-40岁，有稳定收入 |
| **游戏偏好** | 策略、模拟经营、叙事类游戏 |
| **时间特点** | 工作繁忙，碎片时间为主 |
| **消费能力** | 愿意为优质内容付费 |
| **平台习惯** | Web/移动端，偏好即开即玩 |

### 2.2 用户需求分析

| 需求层级 | 需求描述 | 游戏满足方式 |
|----------|----------|--------------|
| **基础层** | 娱乐消遣、打发时间 | 每日晨报阅读体验 |
| **情感层** | 成为故事一部分 | 决策影响世界、被写入小说 |
| **社交层** | 分享独特经历 | 个人传记可分享、赛季排行 |
| **成就层** | 证明智慧与策略 | 多结局、称号系统、排行榜 |

### 2.3 用户场景

**场景一：通勤路上（日常）**
> 小张每天地铁通勤30分钟，打开手机阅读今日晨报。看到苍龙帝国皇子夺嫡有新进展，自己支持的二皇子似乎处于下风。做出一个决策后下线，明天再看结果。

**场景二：周末深度（深入）**
> 周六下午，小李有1小时空闲，深入体验一个事件链。从发现边境走私开始，逐步揭开一个跨国阴谋，最后需要决定向哪方势力告发。

**场景三：赛季冲刺（赛季末）**
> 赛季最后一周，小王为了冲榜，密集登录处理多个紧急事件，与其他玩家竞争赛季排名。

## 三、竞品分析

### 3.1 直接竞品

| 游戏 | 类型 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|--------------|
| **Reigns** | 决策卡牌 | 极简交互、黑色幽默 | 内容有限、缺乏叙事深度 | 更深叙事、持续内容更新 |
| **80 Days** | 叙事冒险 | 精美文字、分支丰富 | 单机、无持续运营 | AI生成无限内容、世界持续运转 |
| **AI Dungeon** | AI叙事 | 无限可能 | 缺乏约束、质量不稳定 | 结构化约束+AI生成，保证质量 |
| **Elyria** (停运) | 沙盒MMO | 永久死亡、真实世界 | 开发难度过大 | 更聚焦叙事、更可控的AI |

### 3.2 间接竞品

| 游戏 | 借鉴点 | 我们的创新 |
|------|--------|------------|
| **十字军之王3** | 宫廷政治、继承系统 | 简化复杂度，强化叙事 |
| **太阁立志传** | 多职业路线、事件驱动 | AI驱动无限事件 |
| **网文小说** | 连载更新、多主角 | 玩家参与改变剧情 |

### 3.3 竞争优势

```
唯一性 = AI叙事 + 世界自主运转 + Web即开即玩 + 低门槛高深度
```

## 四、核心玩法要素

### 4.1 玩法循环

```
┌─────────────────────────────────────────────────────────────┐
│                     核心玩法循环                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │ 阅读晨报 │ → │ 做出决策 │ → │ 看到后果 │             │
│   └──────────┘    └──────────┘    └────┬─────┘             │
│        ↑                               │                    │
│        └───────────────────────────────┘                    │
│                                                             │
│   世界在后台持续运转（Agent系统）                              │
│   玩家是变量，不是世界的中心                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 关键决策类型

| 决策类型 | 示例 | 影响范围 |
|----------|------|----------|
| **效忠选择** | 是否接受某国招揽 | 阵营归属、可用选项 |
| **资源分配** | 援助村民 vs 上缴税收 | 关系、资源、声望 |
| **人际立场** | 帮助A还是帮助B | NPC关系、事件走向 |
| **危机应对** | 逃跑/战斗/谈判 | 生存、声誉、连锁事件 |
| **长期规划** | 投资商业 vs 军事 | 发展路线、后期选项 |

### 4.3 玩家成长路径

```
新手期（第1-7天）
├── 了解世界观和阵营
├── 体验基础决策
└── 建立初始关系网

成长期（第2-4周）
├── 深入事件链
├── 建立声望和影响力
└── 参与阵营事务

成熟期（第1个月+）
├── 影响世界格局
├── 培养势力/继承人
└── 追求赛季目标
```

## 五、设计原则

### 5.1 核心设计哲学

> **"玩家是历史的一部分，不是历史的中心"**

**原则一：叙事优先**
- 任何功能加入前，先问：这个功能能让故事更精彩吗？
- 如果砍掉这个功能，核心体验还能成立吗？

**原则二：AI驱动但可控**
- 使用AI生成内容，但通过模板和约束保证质量
- 关键节点人工设计，填充内容由AI生成

**原则三：低门槛高深度**
- 日常体验简单：读新闻、做选择
- 深度体验丰富：多线叙事、策略规划

**原则四：世界自主运转**
- 没有玩家，世界也会自己运转
- 玩家是变量，不是必需品

### 5.2 设计检查清单

```
任何功能设计时，确认以下问题：

□ 这个功能是叙事必需的吗？
□ 这个功能能在Web上良好运行吗？
□ 这个功能需要复杂的UI吗？
□ 这个功能会增加LLM调用成本吗？
□ 如果砍掉这个功能，核心体验还能成立吗？

如果后三个问题中有超过一个"是"，考虑简化或延后。
```

## 六、成功指标

### 6.1 MVP阶段指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| **日活跃用户（DAU）** | >1000 | 登录统计 |
| **次日留存率** | >40% | 新用户跟踪 |
| **7日留存率** | >20% | 新用户跟踪 |
| **平均每日时长** | >5分钟 | 在线时长统计 |
| **决策完成率** | >70% | 事件参与统计 |
| **NPS评分** | >30 | 用户调研 |

### 6.2 成长期指标

| 指标 | 目标值 | 时间点 |
|------|--------|--------|
| **月活跃用户（MAU）** | >10,000 | 上线后3个月 |
| **7日留存率** | >30% | 上线后3个月 |
| **平均每日时长** | >8分钟 | 上线后3个月 |
| **付费转化率** | >3% | 上线后6个月 |
| **LTV/CAC** | >1.5 | 上线后6个月 |

### 6.3 定性指标

- **叙事满意度**：玩家是否愿意阅读完整晨报？
- **选择有意义感**：玩家是否觉得决策真的影响了世界？
- **分享意愿**：玩家是否愿意分享游戏中的独特经历？

## 七、项目边界

### 7.1 不做的事情

| 范围 | 说明 | 原因 |
|------|------|------|
| **不做3D/复杂视觉** | Web优先，文字为主 | 控制成本，聚焦叙事 |
| **不做实时PvP** | 异步交互为主 | 降低技术复杂度 |
| **不做复杂战斗系统** | 战斗简化为决策结果 | 非核心体验 |
| **不做完整MMO** | 世界按区隔离，非单一世界 | 技术可行性 |
| **不做付费赢** | 付费仅用于装饰/便捷 | 保证公平性 |

### 7.2 MVP范围

**必须实现（P0）**：
- 每日晨报生成与阅读
- 基础决策系统
- 世界状态保存
- 4个开局剧本

**延后实现（P1/P2）**：
- 完整技能树
- 领地建设
- 赛季系统
- 微信小游戏
