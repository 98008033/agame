# Agame 社会运行机制设计

> **核心理念**：游戏本身是一个社会，这个社会需要像真实社会一样运转。NPC的行为有社会逻辑支撑，而非随机；社会变化有因果关系。

---

## 一、现有设计分析

### 1.1 已有系统的优点

| 系统 | 已有设计 | 评价 |
|------|----------|------|
| Agent架构 | 四级分层（Chronos→国家→城邦→NPC） | ✓ 层级清晰，职责分明 |
| NPC生命周期 | 生老病死、继承机制 | ✓ 有动态感，但缺乏社会因果关系 |
| 经济系统 | 双资源（金币+影响力） | ✓ MVP简化合理，但缺乏闭环循环 |
| 关系系统 | 数值化关系(-100~+100) | ✓ 可量化，但缺乏动态变化逻辑 |
| 事件系统 | 事件链、触发机制 | ✓ 有叙事，但事件缺乏社会动力 |

### 1.2 核心不足

**问题1：社会是静态的，缺乏"为什么"**

现有设计中：
- NPC的关系变化基于"玩家互动"，没有基于社会利益的自动变化
- 经济系统是"玩家获取-消费"的单向流动，没有NPC之间的经济循环
- 事件触发基于"条件检测"，而不是社会矛盾的自然爆发

**问题2：NPC是孤立的个体，缺乏社会网络**

现有设计中：
- NPC有自己的属性和技能，但与其他NPC没有利益绑定
- NPC的行为由各自Agent独立决定，缺乏群体行为逻辑
- NPC之间没有竞争、合作、互利的社会关系

**问题3：社会变化缺乏因果链**

现有设计中：
- 战争由Chronos"注入"，而不是矛盾积累爆发
- 政变是预设事件，而不是权力斗争的结果
- 阶层晋升是玩家目标，但社会结构本身是静态的

## 二、社会运行机制总览

### 2.1 社会运转的三大循环

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         社会运转三大循环                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   ┌──────────────┐          ┌──────────────┐          ┌──────────────┐ │
│   │   经济循环    │          │   权力循环    │          │   关系循环    │ │
│   │              │          │              │          │              │ │
│   │ 生产→分配    │          │ 权力→运用    │          │ 关系→影响    │ │
│   │ →消费→再生产 │          │ →争夺→再分配 │          │ →变化→再影响 │ │
│   │              │          │              │          │              │ │
│   └──────────────┘          └──────────────┘          └──────────────┘ │
│          │                         │                         │         │
│          └─────────────────────────┼─────────────────────────┘         │
│                                    │                                    │
│                                    ▼                                    │
│                          ┌──────────────┐                              │
│                          │   社会事件    │                              │
│                          │              │                              │
│                          │ 循环积累→爆发 │                              │
│                          │ →解决→新循环  │                              │
│                          └──────────────┘                              │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 社会动力模型

**核心原理**：社会变化源于矛盾积累和释放。

```
矛盾类型                矛盾来源                爆发形式
─────────────────────────────────────────────────────
资源矛盾    →    分配不均、稀缺竞争    →    军事冲突、贸易战争
权力矛盾    →    权力真空、权力争夺    →    政变、继承危机
关系矛盾    →    利益冲突、背叛猜忌    →    家族恩怨、派系斗争
阶级矛盾    →    上下层利益对立        →    民变、起义、改革
文化矛盾    →    新旧观念冲突          →    宗教改革、传统派反击
```

## 三、社会结构运行机制

### 3.1 阶层系统

**动态阶层设计**：

阶层不是静态的等级标签，而是动态的社会位置，由以下因素共同决定：

```typescript
interface SocialStratum {
  // 阶层定义（非静态）
  tier: 'royal' | 'noble' | 'gentleman' | 'citizen' | 'peasant' | 'outcast';

  // 阶层支撑因素（动态变化）
  factors: {
    economicPower: number;    // 经济实力（资产-负债）
    politicalInfluence: number; // 政治影响力（职位权力+人脉）
    socialNetwork: number;    // 社交网络（重要关系数量+质量）
    reputation: number;       // 声誉（正面标签-负面标签）
    factionSupport: number;   // 阵营支持（阵营声望×效忠程度）
  };

  // 阶层稳定性
  stability: {
    level: number;            // 0-100，越高越稳定
    risks: string[];          // 阶层下降的风险因素
    opportunities: string[];  // 阶层上升的机会
  };
}
```

**阶层判定公式**：

```
阶层等级 = floor((
  经济实力 × 0.3 +
  政治影响力 × 0.3 +
  社交网络 × 0.2 +
  声誉 × 0.15 +
  阵营支持 × 0.05
) / 20)

阶层边界：
- ≥85: 王族
- 65-84: 贵族
- 45-64: 士绅
- 25-44: 市民
- 5-24: 平民
- <5: 流民/罪犯
```

### 3.2 阶层流动机制

**上升通道（每个阵营不同）**：

| 阵营 | 上升通道 | 条件 | 成功率 |
|------|----------|------|--------|
| 苍龙帝国 | 科举 | 学识技能≥L3 + 考试表现 | 30%每年 |
| 苍龙帝国 | 军功 | 战斗技能≥L3 + 参战存活 | 20%每次战役 |
| 苍龙帝国 | 官员提拔 | 建功+上级赏识 | 15%每年 |
| 霜狼联邦 | 决斗胜利 | 战斗技能≥目标 | 40%每次决斗 |
| 霜狼联邦 | 战功积累 | 参战+存活+斩敌数 | 25%每年 |
| 霜狼联邦 | 部族贡献 | 资源贡献≥部族平均值×2 | 10%每年 |
| 金雀花王国 | 商业成功 | 商业技能≥L3 + 资产翻倍 | 30%每周期 |
| 金雀花王国 | 金融投资 | 投资回报率≥150% | 20%每周期 |
| 金雀花王国 | 议会投票 | 购买席位或赢得选举 | 15%每次选举 |
| 边境联盟 | 村民推选 | 声誉≥40 + 村民关系网 | 40%每次推选 |
| 边境联盟 | 建功立业 | 解决村危机 + 声誉≥50 | 25%每次危机 |
| 边境联盟 | 外来招募 | 特殊技能 + 村长认可 | 20%每次招募 |

**下降风险（通用）**：

```
阶层下降触发条件：
- 经济破产：资产<基础消耗×6个月
- 政治失势：被剥夺职位/失去阵营支持
- 社会污名：获得严重负面标签（叛徒、罪犯等）
- 疾病/衰老：健康值<20导致劳动能力丧失
- 战争失败：战败被俘/投降被贬

下降速度：
- 轻度下降：-1阶层，需1个游戏月恢复
- 重度下降：-2阶层，需1个游戏年恢复
- 社会死亡：进入流民/罪犯状态，需特殊事件翻身
```

### 3.3 阶层间的社会张力

**张力产生机制**：

```
阶层张力 = 上层压迫感 - 下层容忍度

上层压迫感：
- 税收负担：税收率 × 下层人口比例
- 资源垄断：上层控制的资源比例
- 权力差距：上层权力总值 vs 下层权力总值

下层容忍度：
- 基础生存保障：粮食/医疗覆盖率
- 流动通道开放度：上升成功案例数量
- 文化认同：传统/宗教的安抚效果

张力阈值：
- 张力<30：社会稳定
- 张力30-50：有怨言但可控
- 张力50-70：可能发生局部抗议
- 张力>70：可能发生民变/起义
```

## 四、经济循环机制

### 4.1 生产-分配-消费闭环

**核心设计**：经济不是玩家的单向积累，而是社会内部的循环。

```
┌─────────────────────────────────────────────────────────────┐
│                    经济循环流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────┐      ┌─────────┐      ┌─────────┐            │
│   │ 生产者   │ ───→ │ 分配者   │ ───→ │ 消费者   │            │
│   │ (农民)   │      │ (商人)   │      │ (全体)   │            │
│   └─────────┘      └─────────┘      └─────────┘            │
│        │                │                │                  │
│        │    ┌───────────┴───────────┐    │                  │
│        │    │                       │    │                  │
│        ▼    │                       │    ▼                  │
│   ┌─────────┐                       ┌─────────┐            │
│   │ 税收者   │ ←──────────────────── │ 劳动者   │            │
│   │ (政府)   │                       │ (全体)   │            │
│   └─────────┘                       └─────────┘            │
│        │                                  │                 │
│        └──────────────────────────────────┘                 │
│                      （再生产投入）                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 NPC经济角色

**生产者NPC**：

| 类型 | 生产内容 | 效率因素 | 产出周期 |
|------|----------|----------|----------|
| 农民 | 粮食 | 土地质量+气候+技能 | 每游戏季 |
| 矿工 | 矿石 | 矿脉品质+工具+体力 | 每游戏月 |
| 铁匠 | 器具/武器 | 技术等级+原材料 | 每游戏周 |
| 药师 | 药剂 | 药材+配方知识 | 每游戏周 |
| 学者 | 知识/书籍 | 学识等级+材料 | 每游戏月 |

**分配者NPC**：

| 类型 | 分配方式 | 利润来源 | 经营范围 |
|------|----------|----------|----------|
| 商人 | 贸易 | 地区差价+手续费 | 跨城/跨国 |
| 行会首领 | 批发 | 会员费+垄断定价 | 本城行业 |
| 银行家 | 金融 | 利息+手续费 | 跨国金融 |
| 税官 | 强制征收 | 税率×人口 | 本国领土 |

**消费者NPC**：

所有NPC都是消费者，消费需求基于：

```
消费需求 = {
  基础需求: {
    粮食: 人口基数 × 生存系数,
    住房: 阶层等级 × 住房标准,
    医疗: 年龄/健康状况 × 医疗需求,
  },

  阶层需求: {
    贵族: 奢侈品 + 仆人 + 府邸维护,
    商人: 商业工具 + 仓储 + 护卫,
    军人: 武器装备 + 马匹 + 军饷,
    学者: 书籍 + 实验材料 + 门徒,
  },

  特殊需求: {
    根据NPC目标和个人偏好，
    如：军事狂人需要更多武器，
    商人需要更多贸易信息，
  }
}
```

### 4.3 经济事件的自然发生

**供需失衡事件**：

```
触发条件：
- 区域粮食供给 < 区域粮食需求 × 0.8
- 连续3个游戏月供给不足

自然后果：
- 粮价上涨（市场调节）
- 商人囤货（投机行为）
- 民怨积累（社会张力增加）
- 可能触发饥荒事件

因果链：
粮食不足 → 粮价上涨 → 贫民买不起 → 商人囤货加剧
→ 民怨爆发 → 抢粮事件 → 政府干预 → 税收调整或赈灾
```

**贸易冲突事件**：

```
触发条件：
- A国对B国商品征收高关税
- A国商人垄断某关键资源

自然后果：
- B国商人利益受损，游说政府
- B国可能报复性征税
- 贸易战开始
- 可能升级为外交危机或战争前兆
```

## 五、权力循环机制

### 5.1 权力分配结构

**权力定义**：权力 = 强制能力 + 合法性 + 资源控制

```typescript
interface PowerStructure {
  // 权力核心
  corePower: {
    military: number;       // 军事力量（士兵数×战力）
    legitimacy: number;     // 合法性（血统+传统+民心）
    resourceControl: number; // 资源控制（税收+矿脉+商路）
  };

  // 权力来源（不同阵营不同）
  powerSource: {
    canglong: ['血统', '科举', '军功', '帝王任命'],
    shuanglang: ['战功', '部族推选', '决斗胜利', '祖灵认可'],
    jinque: ['财富购买', '议会投票', '契约绑定', '商业联盟'],
    border: ['村民推选', '功绩积累', '各方认可', '个人声望'],
  };

  // 权力行使范围
  authority: {
    domain: string[];       // 权力覆盖的领域
    subordinates: string[]; // 下属NPC列表
    dependents: string[];   // 依赖者NPC列表（如被庇护的平民）
  };
}
```

### 5.2 权力争夺的动态机制

**权力真空产生**：

```
权力真空触发条件：
- 关键NPC死亡/失势/被俘
- 继承人空缺或争议
- 新职位设立（如新征服领土需要任命）

权力真空后果：
- 原下属失去庇护，地位动摇
- 潜在继承者开始争夺
- 敌对势力可能趁机渗透
- 社会张力暂时上升
```

**权力争夺过程**：

```
争夺者评估：
每位潜在继承者有一个"继承竞争力"评分：

继承竞争力 = {
  合法性得分：血统距离 × 传统权重 + 制度规定
  实力得分：军事力量 + 经济实力 + 人脉网络
  支持得分：关键NPC的表态支持 × 各NPC权重
  时运得分：时机把握 + 外部环境有利度
}

争夺方式（按阵营）：
- 苍龙：暗中结盟、科举比试、皇帝最终裁决
- 霜狼：决斗挑战、部族投票、战功比拼
- 金雀花：金钱竞选、契约绑定、议会投票
- 边境：村民推选、功绩展示、各方谈判

争夺结果：
- 赢家获得职位，但可能付出代价（如承诺交换）
- 赢家的支持者获得好处（升职/资源）
- 输家的支持者可能被打压或报复
- 争夺过程消耗社会资源（金钱/时间/人心）
```

### 5.3 权力行使的后果反馈

**权力决策的社会反馈**：

```
决策类型        短期后果            长期后果
─────────────────────────────────────────────
加税    →    民怨上升，税收增加    →    阶层张力积累，可能民变
减税    →    民心上升，财政紧张    →    国力下降，可能外敌入侵
征兵    →    兵力增加，劳动力减少  →    生产下降，经济循环受阻
开战    →    资源消耗，领土可能扩张 →    战后重建，权力格局改变
改革    →    新势力上升，旧势力反抗 →    社会重组，可能内战
```

## 六、关系循环机制

### 6.1 NPC社会关系网络

**关系网络结构**：

```typescript
interface SocialNetwork {
  // 关系类型
  relationships: Map<string, Relationship>;

  // 关系强度
  strength: number;         // -100 ~ +100

  // 关系性质
  nature: {
    type: 'kinship' | 'political' | 'economic' | 'social' | 'conflict';
    direction: 'mutual' | 'asymmetric' | 'opposing';
    durability: 'permanent' | 'long-term' | 'short-term' | 'transactional';
  };

  // 关系利益绑定
  bindings: {
    mutualBenefit: string[];  // 共同利益（如：都依赖某资源）
    conflictInterest: string[]; // 冲突利益（如：争夺同一职位）
    debt: string[];           // 债务关系（人情债、金钱债）
    obligation: string[];     // 义务关系（上下级、血缘）
  };
}
```

**关系类型详解**：

| 关系类型 | 形成方式 | 维持条件 | 破裂触发 |
|----------|----------|----------|----------|
| 血缘 | 出生确定 | 家族认同 + 共同利益 | 极端背叛/家族分裂 |
| 婚姻 | 婚礼事件 | 双方维系 + 共同利益 | 死亡/离婚事件 |
| 师徒 | 学习协议 | 技术传承 + 感情纽带 | 背叛/学成离去 |
| 上下级 | 权力任命 | 权力维系 + 互利 | 权力变化/背叛 |
| 商业伙伴 | 契约签订 | 贸易互利 + 信任 | 利益冲突/违约 |
| 政治盟友 | 共同目标 | 目标一致 + 互利 | 目标分歧/背叛 |
| 挚友 | 情感积累 | 情感纽带 + 信任 | 极端背叛 |
| 仇敌 | 冲突积累 | 冲突记忆 + 不信任 | 冲突解决/死亡 |

### 6.2 关系的动态变化机制

**关系变化的驱动力**：

```
关系变化 = Σ(事件影响) + 自然衰减 + 社会压力

事件影响：
- 正面事件：帮助、赠礼、共同成功 → 关系上升
- 负面事件：冲突、背叛、利益对立 → 关系下降

自然衰减：
- 每游戏月不互动：关系 -2
- 超过3游戏月不互动：关系 -5
- 超过1游戏年不互动：关系降至"陌生"基准（±20）

社会压力：
- 身份冲突：如属敌对阵营 → 关系倾向下降
- 公共事件：如公开表态反对 → 关系强制下降
- 舆论压力：如被舆论定性为敌人 → 关系被迫下降
```

**自动关系变化示例**：

```
场景：A和B原本是盟友（关系+60）

1. A被任命为B的竞争对手的上级
   → 身份冲突：关系 -15
   → A被迫表态：关系再 -10
   → B感到背叛：关系再 -5

2. 两人3个月不主动互动
   → 自然衰减：关系 -5

3. 总变化：关系从+60降至+25（从"盟友"变为"认识"）

叙事表现：
- A内心："我也没办法，这是上面的命令..."
- B内心："这家伙果然是两面派，以后小心..."
- 晨报："XX被任命为XX的上司，昔日盟友渐行渐远..."
```

### 6.3 关系影响社会事件

**关系驱动的冲突事件**：

```
触发条件：
- A和B关系< -30（敌对）
- A和B有利益冲突（如争夺同一职位）
- A有行动能力，B有被伤害可能

可能事件：
- A暗中破坏B的计划
- A散布不利于B的谣言
- A向B的上级举报B
- A雇人暗杀B

后果：
- B关系再下降
- B可能报复
- 可能形成"恩怨循环"
- 可能波及双方的关系网络
```

**关系驱动的合作事件**：

```
触发条件：
- A和B关系> +50（友好以上）
- A或B面临困境需要帮助
- 另一方有能力帮助

可能事件：
- A主动提供资源帮助B
- B推荐A获得职位机会
- 双方联合对付共同敌人
- 双方约定商业合作

后果：
- 双方关系上升
- 双方可能形成利益绑定
- 可能形成派系/联盟
- 可能被第三方视为威胁
```

## 八、NPC人生悲欢离合机制

### 8.1 人生事件的社会逻辑

**成功事件**：

```
成功不是"运气"，而是社会因素积累：

成功条件 = {
  个人能力：技能等级 ≥ 挑战难度
  社会支持：关系网络中的支持者数量和质量
  时机把握：社会环境有利于成功（如权力真空期）
  资源支撑：个人资源 ≥ 成功成本
}

成功后果 = {
  阶层上升：根据成功类型调整阶层
  关系变化：盟友关系↑，对手关系↓
  声望变化：获得正面标签，影响力↑
  资源变化：获得相应资源回报
}

叙事表现：
- 晨报：报道成功事迹
- 相关NPC反应：盟友祝贺，对手警惕
- 个人传记：记录人生转折点
```

**失败事件**：

```
失败也有社会逻辑：

失败条件 = {
  个人能力不足：技能等级 < 挑战难度
  社会阻力：对手阻碍或缺乏支持
  时运不济：社会环境不利（如战争期间）
  资源不足：个人资源 < 成本需求
}

失败后果 = {
  阶层风险：可能下降
  关系风险：可能失去盟友信任
  声望风险：可能获得负面标签
  资源损失：投入资源消失
}

叙事表现：
- 晨报：报道失败，可能附带批评
- 相关NPC反应：盟友担忧，对手趁机
- 个人传记：记录挫折和反思
```

### 8.2 婚姻系统的社会机制

**婚姻不是"选择"，而是社会匹配**：

```
婚姻匹配度 = {
  阶层匹配：双方阶层差距 ≤ 2级
  家族利益：双方家族是否有互利需求
  个人意愿：双方关系值 ≥ 40
  社会认可：社会舆论是否支持（如阵营关系）
}

婚姻类型：
- 政治联姻：家族主导，个人意愿次要
- 商业联姻：财富结合，互利主导
- 情感婚姻：个人意愿主导，家族次要
- 强制婚姻：权力强迫，违背意愿

婚姻后果：
- 双方关系绑定（通常升至+60以上）
- 双方家族关系变化（联姻+20，或冲突恶化）
- 财产合并/分割
- 可能产生子嗣
- 社会地位变化（如联姻提升阶层）
```

### 8.3 丧亲系统的社会影响

**死亡不仅是个人事件，更是社会事件**：

```
NPC死亡的社会影响：

对家族：
- 配偶失去伴侣，可能再婚或守寡
- 子女失去父母，继承关系激活
- 家族失去成员，力量和声望调整

对权力：
- 职位空缺，触发继承/争夺
- 下属失去庇护，地位动摇
- 敌对势力可能趁机

对关系网络：
- 死者的盟友失去支持
- 死者的仇敌失去目标（但仇恨可能转移）
- 死者的债务和义务需要处理

对社会：
- 晨报报道（重要人物）
- 相关事件触发（如继承危机）
- 社会格局调整
```

### 8.4 晚年与传承机制

**晚年NPC的特殊状态**：

```
晚年状态（年龄>60）：
- 能力衰退：战斗/劳动能力下降
- 经验积累：谋略/外交能力可能上升
- 健康风险：自然死亡概率增加
- 传承压力：需要安排继承

传承机制：
- 指定继承人：明确指定（通常子女或弟子）
- 培养继承人：传承技能和资源
- 分配遗产：财产、职位、人脉的分配
- 安排后事：遗嘱、遗言、托付

传承事件：
- 临终托付：指定继承人和关键嘱托
- 遗产分配：财产和权力的正式转移
- 继承仪式：继承人正式接任
- 家族重组：权力结构重新调整
```

### 9.2 矛盾爆发检测算法

```typescript
function detectConflictExplosion(
  dailyUpdate: DailySocialUpdate
): ConflictEvent[] {
  const events: ConflictEvent[] = [];

  // 1. 检测资源矛盾爆发
  for (const [resourceId, conflictValue] of dailyUpdate.conflicts.resourceConflicts) {
    const threshold = getConflictThreshold('resource', resourceId);
    if (conflictValue >= threshold) {
      const event = generateResourceConflictEvent(resourceId, conflictValue);
      events.push(event);
    }
  }

  // 2. 检测权力矛盾爆发
  for (const [positionId, conflictValue] of dailyUpdate.conflicts.powerConflicts) {
    const threshold = getConflictThreshold('power', positionId);
    if (conflictValue >= threshold) {
      const event = generatePowerConflictEvent(positionId, conflictValue);
      events.push(event);
    }
  }

  // 3. 检测关系矛盾爆发
  for (const [relationshipId, conflictValue] of dailyUpdate.conflicts.relationshipConflicts) {
    const threshold = getConflictThreshold('relationship', relationshipId);
    if (conflictValue >= threshold) {
      const event = generateRelationshipConflictEvent(relationshipId, conflictValue);
      events.push(event);
    }
  }

  // 4. 检测阶层张力爆发
  for (const [factionId, tensionValue] of dailyUpdate.conflicts.stratumConflicts) {
    const threshold = getConflictThreshold('stratum', factionId);
    if (tensionValue >= threshold) {
      const event = generateStratumConflictEvent(factionId, tensionValue);
      events.push(event);
    }
  }

  // 按重要性排序，返回最严重的若干事件
  return events
    .sort((a, b) => b.severity - a.severity)
    .slice(0, getMaxDailyEvents());
}
```

### 9.3 事件后果反馈算法

```typescript
function applyEventConsequences(
  event: ConflictEvent,
  worldState: WorldState
): WorldState {

  // 1. 矛盾释放/加剧
  const conflictRelease = event.conflictReleased || event.conflictIntensified;
  worldState.conflicts[event.conflictType] -= conflictRelease;

  // 2. 资源变化
  for (const change of event.resourceChanges) {
    worldState.resources[change.resourceId] += change.delta;
  }

  // 3. 权力变化
  for (const change of event.powerChanges) {
    worldState.npcs[change.npcId].power += change.delta;
  }

  // 4. 关系变化
  for (const change of event.relationshipChanges) {
    const relation = worldState.relationships[change.from][change.to];
    relation += change.delta;
    // 可能触发新的关系矛盾
    if (relation < -30) {
      worldState.conflicts.relationshipConflicts[`${change.from}-${change.to}`] += 10;
    }
  }

  // 5. 阶层张力变化
  for (const change of event.stratumTensionChanges) {
    worldState.conflicts.stratumConflicts[change.faction] += change.delta;
  }

  // 6. NPC状态变化（如伤亡、职位变化）
  for (const change of event.npcStateChanges) {
    applyNPCStateChange(worldState.npcs[change.npcId], change);
  }

  return worldState;
}
```

## 十二、设计验证清单

**每个社会机制加入前，问自己：**

1. 这个机制是否让社会看起来"自己运转"？
2. NPC的行为是否有社会逻辑支撑（而非随机）？
3. 事件是否是矛盾积累的自然爆发（而非凭空注入）？
4. 社会变化是否有因果关系（而非无源之变）？
5. 这个机制能否在叙事中体现（让玩家感知）？
