### 2.3 Agent架构 (docs/agent-architecture.md)

#### ✅ 与叙事系统一致性

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 叙事生成接口 | ✅ 通过 | 世界级Agent (Chronos) 的职责包含"重大事件叙事文本" |
| 记忆管理 | ✅ 通过 | 分层记忆设计与叙事系统的四层记忆架构一致 |
| 事件传递 | ✅ 通过 | 事件总线支持 `world:event_generated` 等叙事相关事件 |

#### ⚠️ 建议改进

1. **叙事专用输出格式**
   ```typescript
   // 建议在第2.1节的 AgentPlan 接口中增加
   interface NarrativeOutput {
     format: 'novel_chapter' | 'news_article' | 'dialogue';
     style: 'historical' | 'dramatic' | 'neutral';
     wordCount: { min: number; max: number };
     perspective: 'omniscient' | 'character' | 'player';
   }
   ```

2. **Chronos叙事职责增强**
   ```
   当前Chronos职责：
   - worldBalance
   - historyProgression
   - crossCampEvents

   建议增加：
   - narrativeWeaving: 将碎片事件编织成连贯叙事
   - storyHookGeneration: 为下一章生成悬念
   ```

3. **Prompt与叙事模板衔接**
   ```
   当前：Agent Prompt使用三层设计 (L1系统/L2场景/L3任务)
   建议：与叙事系统的Prompt模板统一
   例如：worldAgentPrompt.scenario 应使用与小说生成相同的上下文格式
   ```

#### ❌ 发现问题

**问题1**: 记忆管理系统缺乏叙事长期记忆的具体实现
```
位置: 第1.2节 StatePropagation
现状: 只有 world/nation/city 三层状态快照
缺失: 叙事记忆（如"上一章的悬念"）的传递机制
建议: 增加 narrativeMemory 字段
```

**问题2**: 成本估算未考虑叙事生成
```
位置: 第3.2节 成本对比分析
现状: 只计算了Agent运行的token成本
缺失: 每日小说章节(3000-5000字)的生成成本
建议: 根据叙事系统文档补充成本估算
```
